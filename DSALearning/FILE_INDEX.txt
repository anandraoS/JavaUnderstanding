    For: Directed acyclic graphs (DAGs) only
    DRY RUN: Shows post-order traversal using stack

9️⃣ RecursionAndDP.java (450+ lines)
─────────────────────────────────────

PURPOSE: Master recursion and dynamic programming
WHEN: Week 6-7 of study plan

RECURSION CONCEPTS:
  Base case (must have!)
  Recursive case
  Often leads to exponential time

DYNAMIC PROGRAMMING:
  Three requirements:
    1. Overlapping subproblems
    2. Optimal substructure
    3. Can memoize results

  Two approaches:
    Memoization (Top-Down): Recursive + cache
    Tabulation (Bottom-Up): Iterative + table

ALGORITHMS (6):
  ✓ Fibonacci - 3 approaches
    Pure Recursion: O(2^n) - VERY SLOW
    Memoization: O(n) - Much better
    Tabulation: O(n) - Clean and iterative
    Space Optimization: O(1) - Just two variables
    DRY RUN: Shows each approach

  ✓ Coin Change - Time: O(amount * coins), Space: O(amount)
    Problem: Minimum coins for amount
    dp[i] = minimum coins for amount i
    DRY RUN: Shows filling DP table

  ✓ Longest Common Subsequence - Time: O(m*n), Space: O(m*n)
    Problem: Find longest common sequence (not contiguous)
    2D DP table: dp[i][j] = LCS of first i and j chars
    DRY RUN: Shows 2D table filling

  ✓ 0/1 Knapsack - Time: O(n*capacity), Space: O(n*capacity)
    Problem: Maximum value with weight limit
    Each item used 0 or 1 time
    DRY RUN: Shows table building and decisions

KEY INSIGHTS:
  • Identify overlapping subproblems
  • Memoization easier to implement
  • Tabulation more efficient
  • Many problems reduce exponential to polynomial!
